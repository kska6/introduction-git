# CLIでファイルを編集する手順の指示対象対応表

| 出典 | 目的 | 具体対象 | 役割 | 前後関係 | 候補語 | 初出定義 |
|---|---|---|---|---|---|---|
| ユーザー追加指示「コマンド例を提示する時には手違いないようにカレントディレクトリの確認から作業が始まるようにする」および訂正「git --version, git configはグローバル環境での作業のためpwdは不要」 | カレントディレクトリに依存する操作だけ、意図しない場所で実行する手違いを防ぐ | 現在のリポジトリやファイルを対象とするコマンド例は最初に現在の作業ディレクトリを表示し、カレントディレクトリに依存しない `git --version` と `git config --global` の例はそのコマンドから始める構成 | 開始条件 | カレントディレクトリへの依存有無を区別した後、依存する例は「カレントディレクトリの確認」から始め、依存しない例は対象コマンドから始める | カレントディレクトリ確認の適用条件 | カレントディレクトリ確認の適用条件とは、現在のリポジトリやファイルを対象とするコマンド例だけを `pwd` から始め、カレントディレクトリに依存しない `git --version` と `git config --global` の例は対象コマンドから始める基準を指す。 |
| ユーザー指示「カレントディレクトリ確認 pwd」 | 意図した場所で作業を始める | シェルが現在の作業場所として扱っているディレクトリの絶対パスを表示する操作 | 事象 | 「pwd」の後に「cd」 | `pwd` | `pwd` は、シェルが現在の作業場所として扱っているディレクトリの絶対パスを表示するコマンドである。 |
| ユーザー指示「ディレクトリ移動 cd」 | 編集対象があるディレクトリへ移る | シェルの現在の作業場所を、指定したリポジトリのディレクトリへ変更する操作 | 事象 | 「pwd」の後、「ls」の前 | `cd` | `cd` は、シェルの現在の作業場所を指定したディレクトリへ変更するコマンドである。 |
| ユーザー指示「ファイル確認 ls」 | 編集前に対象ファイルと周辺のファイルを確認する | 現在のディレクトリにあるファイル名とディレクトリ名を一覧表示する操作 | 事象 | 「cd」の後、「ファイル編集」の前 | `ls` | `ls` は、現在のディレクトリにあるファイル名とディレクトリ名を一覧表示するコマンドである。 |
| ユーザー指示「ファイル編集 nano/vim/emax」 | ターミナル内で対象ファイルを作成または変更する | `nano`、`vim`、`emacs` のいずれか一つへ対象ファイルのパスを渡し、編集して保存する操作 | 手段 | 「ls」の後、「less」の前 | CLIエディタによるファイル編集 | CLIエディタによるファイル編集とは、`nano`、`vim`、`emacs` のいずれか一つをターミナルで実行し、対象ファイルを作成または変更して保存する操作を指す。 |
| ユーザー指示「ファイル内容の表示 cat」および訂正「ファイル内容の表示にはlessを使う」 | 保存した内容が意図どおりか確認する | 編集して保存したファイルの内容を `less FILE` で表示し、`q` で終了する操作 | 事象 | 「ファイル編集」の後に「less」 | `less` | `less` は、指定したファイルの内容をターミナルに表示し、`q` で終了するコマンドである。 |
