# Git 比喩候補の指示対象対応表

この対応表は、比喩の名称を先に決めず、Git の各概念が実際に指す対象と役割を固定するための基準である。

## 変更の一部を対話的にステージする流れ

| 出典 | 目的 | 具体対象 | 役割 | 前後関係 | 候補語 | 初出定義 |
|---|---|---|---|---|---|---|
| ユーザー追加指示「git add -p/-i」、Git公式資料 | 一つのファイルに混在する変更を目的ごとに分ける | インデックスとワークツリーの差分から、近接する変更行を一つの選択単位として表示したもの | 値 | 「git diff」から「hunkの表示」、その後「ステージするか選択」へ | hunk | hunkとは、差分の中で近接する変更行をまとめ、対話的に選択できる単位を指す。 |
| ユーザー追加指示「git add -p」、Git公式資料 | 次のコミットへ含める変更だけを選ぶ | 追跡済みファイルのhunkを順に表示し、利用者の回答に応じて選んだ部分だけをインデックスへ反映する操作 | 事象 | 「git add -p」から「y/n/s/qで選択」、その後「git diff --staged」へ | patchモード | patchモードとは、追跡済みファイルの差分をhunkごとに確認し、選んだ部分だけをステージする操作を指す。 |
| ユーザー追加指示「git add -i」、Git公式資料 | 対話メニューからステージ操作を選ぶ | status、update、revert、add untracked、patch、diffなどの項目を表示し、選択した操作を実行するメニュー | 手段 | 「git add -i」から「メニュー項目の選択」、必要に応じて「patch」へ | interactiveモード | interactiveモードとは、対話メニューからステージ対象の更新、取り消し、patch選択、差分確認などを選ぶ操作方法を指す。 |
| ユーザー追加指示「git add -p/-i」、Git公式資料 | 意図した部分だけが次のコミットへ入ることを確認する | 操作後にステージ済み差分と未ステージ差分を別々に表示し、同じファイルの変更が分かれている状態を確認する操作 | 事象 | 「部分ステージ」から「git diff --staged」と「git diff」、その後「commit」へ | 部分ステージ後の確認 | 部分ステージ後の確認とは、選んだ変更と残した変更をそれぞれ差分表示し、次のコミットへ含める内容を確定する操作を指す。 |
