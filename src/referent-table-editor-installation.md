# テキストエディタ導入節の指示対象対応表

| 出典 | 目的 | 具体対象 | 役割 | 前後関係 | 候補語 | 初出定義 |
|---|---|---|---|---|---|---|
| ユーザー指示「『手元環境への導入』で今後のテキスト編集に用いるnanoのインストールセクションを設ける」、訂正「Windowsへのnanoインストールはwingetを既定の方法とする」および指定「GNU.nano」 | 後続の演習でファイルを編集する前に、利用者が実行できるエディタを準備する | 「手元環境への導入」の中で、Nano の有無を確認し、Windowsでは `winget install GNU.Nano` を既定の導入手段として、macOSとDebian / Ubuntuでは各OSのパッケージ管理コマンドで導入し、起動できることを確認する手順 | 開始条件 | 「手元環境への導入」の後続で行うテキスト編集より前 | テキストエディタの事前準備 | テキストエディタの事前準備とは、後続の演習へ進む前に、Nano の有無を確認し、Windowsでは `winget install GNU.Nano` を既定の手段として導入し、起動できる状態にする手順を指す。 |
| ユーザー指示「この中に vim/emacs/code(VSCode)/edit(for Windows) の選択肢も提示しておく」 | 利用環境や操作経験に合うエディタを一つ選べるようにする | 後続のファイル編集で、Nano、Vim、ターミナル内の Emacs、CLIから起動する VS Code、Windows 11 の Edit のいずれか一つを使う選択肢 | 手段 | Nano の導入手順と同じ節で提示し、後続のテキスト編集では選んだ一つを使う | 利用するテキストエディタ | 利用するテキストエディタとは、後続のファイル編集で使う Nano、Vim、ターミナル内の Emacs、CLIから起動する VS Code、Windows 11 の Edit のうち、利用者が選んだ一つを指す。 |
| ユーザー指示「git editorの既定をnanoにするconfigも実行」 | Gitが対話的な編集に使うエディターを、導入済みのNanoに統一する | `git config --global core.editor nano` を実行して設定するグローバルGit設定 | 環境設定 | Nanoの導入後に続く「Gitの初期設定」で、名前、メールアドレス、既定ブランチと合わせて実行する | Gitの既定エディター設定 | Gitの既定エディター設定とは、Gitがコミットメッセージなどの対話的な編集に使うコマンドを、グローバル環境でNanoに指定する設定を指す。 |
